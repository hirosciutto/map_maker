# 地理圏レイヤー(レイヤー2)仕様

- 版: v0.3(2026-07-14。**14圏→30圏**へ更新 = blockland design v2.09 §18.5.1。line36 の誤コード `EAS` を修正)
- 旧版: v0.2(2026-07-12。14圏。design v2.05)
- 目的: Blockland の生息域仕様「**地理圏 × バイオーム**」(blockland/docs/design.md §18.4/§18.5.1)のための `zone_map.json` を、本ツールで塗って出力できるようにする。
- 発注元: blockland design TBD #65。

## 1. 概念

既存のバイオームマップ(レイヤー1)の**上に重なる第2のグリッド**。フルのレイヤーシステムは不要で、必要なのは次の3点だけ:

1. **下敷き表示**: レイヤー2 編集中は、レイヤー1(バイオーム)を**淡色(減光)で下敷き表示**する。大陸の形をなぞって塗るための参照。
2. **海マスク**: レイヤー1 が水域バイオーム(`colorFamily === "ocean_deep"` = `OCN`/`SHF`/`DPO`/`TRN`)のセルは**暗くマスクされ、塗れない**(既存のマスク機構の流用を想定)。
3. **地理圏パレット**: レイヤー2 編集中はパレットが**地理圏30色**に切り替わる。

## 3. 塗りのルール

- **粒度は粗くて良い**。バイオームのような1pxの精密さは不要(目的はゲーム上の生息分けであり、境界の厳密な地理ではない)。
- **全陸セル(水域バイオーム以外)が塗られていること**が完成条件。未塗り陸セルは書き出し時に**検証エラー**にする。
- 島は最寄り圏に含める(例: マダガスカル=`MDG`、日本=`JPN`、ニュージーランド=`NZL`、グリーンランド=`ARC`)。
- **島嶼固有圏の面積確保 [design §18.5.1 申し送り]**: `MDG` `NZL` `JPN` `CAR` `NGU` `WLC` は小面積のため、**実在の島マスへ確実に割り当てる**(この世界は架空なので任意の島を「マダガスカル相当」と宣言してよい)。ぴったりの島が無ければ最寄りの島を充てるか、その圏はこの世界に無しとする。マップ差し替え時はこれらの島に意味ある面積を確保する。
- 北極圏(`ARC`)・南極圏(`ANT`)は緯度帯ではなく「極地の陸・氷床」を塗る(グリーンランド、シベリア最北縁、南極大陸)。

## 4. 出力(`zone_map.json`)

- `biome_map.json` と**同一グリッド・同一フォーマット**(固定3文字コードの `rows` 連結+`legend`)。別ファイルとして出力する。
- **海セルの自動割当**: 書き出し時に、水域セルへ**最も近い塗り済み陸セルの地理圏**を自動で書き込む(BFS/距離変換。等距離はセル座標の辞書順で決定論に)。→ 人間は海を一切塗らなくてよい。
- `legend` には各圏の `code / name / jp / color` を含める。

## 5. 実装メモ(既存機構の流用)

- グリッド: `state.grid` と同構造の `state.zoneGrid` を追加(保存・読込も同型)。
- モード切替: 「レイヤー1(バイオーム)/レイヤー2(地理圏)」のトグル。レイヤー2 では**描画先グリッドとパレット定義を差し替えるだけ**で、ペイント(`paintCell`)・塗りつぶし(`floodFill`)・PNG/JSON入出力(`exportPng`/`exportJson`/`importJson`)のコードパスは共通化できる想定。
- 水域判定はレイヤー1 を参照(リアルタイム。レイヤー1 で海岸線を直した場合はレイヤー2 の該当セルを未塗りに戻す)。
- 読込互換: `zone_map.json` が無い旧プロジェクトはレイヤー2 を空で開く。
- undo/redo(`state.undoStack`)は現在アクティブなレイヤーのグリッドに対して働くこと。
