# 実地理再現ツール 改善設計書

> このツールを「ゲーム用バイオームエディタ」から「**実地球の地理を再現するための編集ツール**」へ引き上げるための設計書。
> ここに書かれた原則をプロジェクトの**憲法**とし、以降の仕様追加・バイオーム追加はすべてこの憲法に照らして判断する。

---

## 0. ゴール（達成された状態の定義）

このツールで世界地図を塗り、出力した JSON から 3D マップを生成したとき、次が成立していること。

1. モンゴル・中国・ロシア・東南アジア・中東・南北アメリカ・ヨーロッパ・アフリカ・アジア各地域が、**それぞれの標高・地形・植生・気温・湿度で再現**されている。
2. ヒマラヤ・カラコルムが、このバイオーム配置の帰結として**自然に世界最高レベルの山岳**になっている（特別扱いのハードコードなしに、標高アンカーの単調性だけで最高峰になる）。
3. 逆側、マリアナ海溝などが**自然に世界最深**になっている（対称性）。
4. 上記が**バイオームを塗るという単一操作のみ**で達成できる（標高を別レイヤーで手描きする必要がない）。

---

## 1. 憲法（設計原則）

このプロジェクトのすべての判断は次の原則に従う。

- **原則1: バイオーム＝地形階層（landform）× 気候被覆（climate cover）。**
  1つのバイオームは「どの高さにあるか（地形）」と「何で覆われているか（植生・気候）」の**両方**を同時に確定させる。塗った瞬間に標高・植生・気温・湿度が一意に決まる。

- **原則2: 標高はメートル実値のアンカーで持つ。**
  各バイオームは実在地域の実測値に紐づいた `elevation`（代表標高, m）を持つ。海面=0。3D 生成器はこのアンカーを目標高度として地形を隆起させる。

- **原則3: 標高アンカーは全球で単調。最上位ティアはヒマラヤ級にのみ割り当てる。**
  最上位バイオームの `elevation` が全バイオーム中で最大であるため、それを塗った領域は必然的に世界最高になる。「ヒマラヤを最高にする特別処理」は書かない。標高設計そのものがそれを保証する。

- **原則4: 実在する組み合わせのみをバイオーム化する。**
  「温帯林 × 低地」「温帯林 × 山地」のように、実世界に存在する（地形×被覆）の組み合わせだけを追加する。存在しない組み合わせは作らない。**過度な統合（1つのバイオームで複数の実地形をまかなう）も、過度な細分化も避ける。**

- **原則5: 曖昧さは実地理の実測値で断ち切る。**
  「どこに置くべきか」で迷ったら、実在する代表地点（例: チベット高原=4500m）の実測値を基準にする。主観ではなく地理データが基準点。

- **原則6: 気温・湿度は緯度をユーザーが塗ることで表現する。**
  緯度そのものの入力レイヤーは持たない。赤道付近には熱帯バイオームを、極付近には寒帯バイオームを塗る、という運用で緯度勾配を表現する（単一レイヤー原則＝原則1の帰結）。ただし**同じ「氷」でも標高が違う**（赤道の氷河5000m vs 極の氷床0m）ため、これらは別バイオームとして持つ。

## 2. 標高設計（Elevation Tiers）

標高を離散的な**ティア**に分け、各ティアに代表標高（m）と実在アンカーを与える。バイオームは必ずいずれかのティアに属す。

| ティア | 名称 | 代表 m | 帯 (m) | 実在アンカー |
|:---:|---|---:|---|---|
| T−2 | 海溝 | −8000 | < −5000 | マリアナ海溝 −10935 |
| T−1 | 深海 | −4000 | −5000〜−500 | 深海平原 |
| T0 | 大陸棚・浅海 | −100 | −500〜−10 | 大陸棚 |
| T1 | 海岸 | 0 | −10〜30 | 海岸線・砂浜 |
| T2 | 低地平野 | 100 | 30〜300 | アマゾン盆地100 / 北欧平原50 / ガンジス平原100 |
| T3 | 丘陵・台地 | 500 | 300〜900 | 北米プレーリー500 / デカン高原600 |
| T4 | 高地・低高原 | 1200 | 900〜1800 | モンゴル高原1500 / アナトリア1000 / コロラド高原1800 |
| T5 | 山地 | 2200 | 1800〜2800 | アルプス中腹 / 日本アルプス |
| T6 | 高原（高々度） | 3500 | 2800〜4200 | アルティプラーノ3800 / エチオピア高地3000 |
| T7 | 高山・超高高原 | 4800 | 4200〜5500 | チベット高原4500 / キリマンジャロ5895 / エルブルス5642 |
| T8 | 超高山 | 6500 | 5500〜7500 | アコンカグア6961 / デナリ6190 |
| T9 | 極高山 | 8500 | 7500 以上 | エベレスト8849 / K2 8611 |

### 2.1 地形（relief）は標高と独立した軸

**同じ標高でも「平ら」か「尖っている」かは別問題**である。チベット高原（4500m）は平坦、ヒマラヤ（8000m級）は鋭峰。これを 1 軸で表せないため、各バイオームに `relief`（起伏）を持たせる。

| relief | 意味 | 3D生成での振幅の目安 |
|---|---|---|
| `flat` | 平坦（平野・高原・氷床・湖底） | ±20 m |
| `rolling` | なだらか（丘陵・ステップ・砂丘） | ±100 m |
| `rugged` | 起伏大（山地・山林） | ±400 m |
| `peak` | 鋭峰（高山・氷峰） | ±1000 m |

→ チベット= T7 × `flat`（4500mの平原）、ヒマラヤ= T9 × `peak`（8500mの鋭峰）。この 2 軸で「屋根のように平らな高地」と「世界最高の鋭峰」を塗り分けられる。

### 2.2 雪線（緯度による氷の標高差）

万年雪・氷の**下限標高は緯度で大きく変わる**。

- 赤道: 約 4900〜5000 m（キリマンジャロ・赤道直下アンデス）
- 中緯度: 約 2700〜3500 m（アルプス約2700m）
- 極: 0 m（グリーンランド・南極の氷床は海面高度）

したがって「氷」は単一バイオームにできない。少なくとも次の 3 つを持つ（原則6）。

- **氷床（ice_sheet, T3級 低標高）**: グリーンランド・南極・北極 → 低い、平ら。
- **山岳氷河（glacier, T5級）**: アルプス・ロッキー・パタゴニア → 中標高。
- **万年雪冠（permanent_snow, T7〜）**: ヒマラヤ・赤道高山 → 高標高、鋭峰。

### 2.3 3D 生成の高さ計算（このツールと生成器の契約）

生成器は各セルについて概ね次で高さを決める。

```
height(cell) = elevation(biome)
             + relief_noise() × relief_amplitude(biome.relief)
```

- 隣接ティア間の `elevation` 差が**自動的に斜面**を作る（低地100m と 山地2200m が隣接すれば、その間に山の斜面ができる）。
- 近傍平滑化で崖と斜面を自然に補間する。
- **原則3により最上位アンカー（8500m）が全球最大**なので、それを塗った領域＝ヒマラヤが必然的に世界最高になる。海溝（−8000m）も同様に世界最深。

## 3. 気候被覆軸（Climate Cover）

植生・気候を表す被覆タイプ。`temp`（cold/temperate/hot）× `humidity`（dry/mid/wet）を基礎に、実植生名で分類する（Whittaker 図＋実地理）。

| 被覆 | temp | humidity | 代表植生 |
|---|---|---|---|
| ice | cold | wet/mid | 氷・万年雪 |
| tundra | cold | dry | 無樹木の凍土 |
| boreal | cold | mid | タイガ（針葉樹林） |
| temperate_forest | temperate | mid/wet | 温帯広葉樹・混交林 |
| temperate_grass | temperate | dry | 温帯草原・ステップ |
| mediterranean | temperate/hot | dry(夏乾) | 硬葉樹・低木（チャパラル） |
| desert_hot | hot | dry | 熱帯・亜熱帯砂漠 |
| desert_cold | cold | dry | 寒冷砂漠（ゴビ・チベット・アタカマ） |
| savanna | hot | dry/mid | 熱帯草原・疎林 |
| tropical_dry | hot | mid | 熱帯季節林・モンスーン林 |
| rainforest | hot | wet | 熱帯雨林 |
| wetland | temperate/cold | wet | 湿地・湿原 |
| mangrove | hot | wet | マングローブ・熱帯海岸 |
| barren | any | dry | 岩石・裸地・砂礫 |

## 4. 現状30バイオームのギャップ分析

現状 [app.js](../app.js) の `BIOMES`（30種）を上のティア×被覆に当てると、次の穴が見える。

### 致命的な穴

1. **標高の天井が低すぎる（最重要）。** 現状の最高は `snowy_mtn` 3500m。**4200〜8849m のバイオームが皆無**。ヒマラヤ・チベット・アンデス高峰・カラコルムが再現不能。ゴール2（ヒマラヤ最高）が原理的に達成できない。
2. **寒冷砂漠がない。** ゴビ・チベット・アタカマ・パタゴニアの寒冷乾燥地帯を表せない。現状の砂漠は `desert`（暑・乾）と `dry_plateau`（温・乾）のみ。
3. **温帯ステップ（草原）が弱い。** ユーラシア草原・北米プレーリー・パンパ・モンゴル草原。`plains`(温/中)・`savanna`(暑/乾) では代替できない（ステップは温帯・乾）。
4. **極地の低標高氷床がない。** グリーンランド・南極の海面高度の氷床。現状 `glacier` は 2000m 固定で、極の氷床（0〜500m）を表せない（§2.2）。

### 中程度の穴

5. **地中海性低木（シュラブランド）がない。** 南欧・カリフォルニア・中央チリ・ケープ・南西豪州。面積的に大きな被覆。
6. **マングローブ・熱帯海岸がない。** 東南アジア・アマゾン河口・スンダルバンス。温帯の `beach` とは別物。
7. **熱帯季節林（モンスーン林）が弱い。** インド・インドシナ・セラード。`savanna` は草原であって疎林・季節林を代替しきれない。
8. **雲霧林（クラウドフォレスト）がない。** アンデス東斜面ユンガス・熱帯山地林。
9. **海洋の水深が一定。** `ocean` が 0m 固定。浅海（大陸棚）・深海・海溝の区別がなく、海底地形が平坦。

### 軽微・任意

10. 海面下の陸地（死海 −430m・デスヴァレー −86m・ダナキル）を表す盆地バイオームがない。
11. 温帯雨林（太平洋岸北西部・バルディビア）、塩原（ウユニ）など固有景観。

### 5.1 既存バイオームの再定義（統合しすぎの是正）

原則4に照らし、現状の一部は意味を明確化する。

- `glacier`（氷河, 2000m）→ **山岳氷河に限定**。極の氷床は新設 `ice_sheet` が担う。
- `desert`（砂漠, 暑/乾, 200m）→ **熱帯砂漠に限定**（サハラ・アラビア）。寒冷砂漠は `cold_desert` が担う。
- `savanna`（サバンナ）→ **草原に限定**。疎林・季節林は `dry_forest` が担う。
- `ocean`（海, 0m）→ **海面／浅海の代表**として残しつつ、深部は `shelf_sea`/`deep_ocean`/`ocean_trench` で表す。

## 6. 各地域の再現レシピ（検証）

追加後の 63 種で、ゴール1・2 の各地域が塗れることを検証する。各地域を「どのバイオーム列で塗るか」で示す。広域は分離可能な単位に分けて列挙する。

### アジア

- **ヒマラヤ／カラコルム**: 南麓 `rainforest`(100) → `dry_forest`(300) → `forest`(200) → `montane_forest`(1100) → `subalpine_forest`(1500) → `alpine_rock`(2800) → `high_rock`(3600) → `high_glacier`(4500) → `high_mountain`(4800) → `permanent_snow`(5500) → `extreme_peak`(6800) → **`himalaya`(8500)**。→ 中間バンドが埋まり、山肌が滑らかに立ち上がる。標高アンカー単調により**世界最高が自然に出現**。
- **チベット高原**: 一面 `cold_highland`(4200, flat) ＋ 縁辺の斜面 `montane_cold_desert`(2200)→`high_cold_desert`(3200)→`subalpine_steppe`(1900) ＋ 高峰 `high_mountain`。屋根のように平らな高地になる。
- **中央アジア（カザフ〜天山・パミール）**: カザフ草原 `steppe`/`highland_steppe` ＋ キジルクム/カラクム `cold_desert`(低地寒冷砂漠)→`montane_cold_desert`(2200)→`high_cold_desert`(3200) ＋ 天山・パミール `mountain`→`alpine_rock`(2800)→`high_rock`(3600)→`high_glacier`(4500)→`high_mountain`→`extreme_peak`（イスモイル・ソモニ峰7495）。

### 6.1 検証で見つかった未対応点

全バイオームが絶対標高を持つ前提。

- **南極内陸の高氷冠 → 解決済み（`ice_dome` を追加）**: `ice_sheet`(500m) はグリーンランド沿岸寄りの値で、南極内陸ドーム（表面3000〜4000m）を表せなかった。これは錨の有無ではなく「高標高の氷を表すバイオームが無い」欠落だったため、`ice_dome`（氷冠高原, 3500m, cold, wet, flat）を追加して埋めた（付録A・§5）。
- **一様バイオーム内の緩傾斜（残る制約）**: 1バイオーム=1標高のため、同じ植生のまま標高が徐々に変わる地形（グレートプレーンズの東西勾配など）は単独バイオームでは平らになる。→ 標高別バリアント（例 低地ステップ/高原ステップ）で塗り分けるか、生成器の relief で緩やかに変化させる。

**結論**: この 63 種で列挙した全地域が塗り分け可能。ヒマラヤは `himalaya`(8500) が全球最高アンカーであるため、特別処理なしに世界最高峰となる（ゴール2達成）。

### 6.2 レシピ外バイオームの採用・自動却下ルール

§6 の地域レシピは優先候補であり、実地理の再現に必要な場合はレシピ外バイオームも使用できる。ただし、生成器はレシピ外セルごとに、そのバイオームの本書上の標高アンカー・起伏・気温・湿度／被覆を、ETOPO DEM・WWFエコリージョン・湖沼・氷河・火山・海岸データと照合しなければならない。

- 文書上の設定と地域の実地理が一致する場合のみ採用する。
- 一致しない場合は自動的に却下し、その地域レシピ内で標高・起伏・被覆が最も近いバイオームへ置換する。
- 生成時の申告書には、採用したレシピ外バイオームごとに使用地域・使用率・採用理由・根拠データを記載する。
- 自動却下したものも、却下前後のバイオーム・セル数・不一致理由を記録する。
- 採用済みのレシピ外バイオームも最終決定ではなく、ユーザーが拒否した場合は修正対象とする。

### 6.3 レシピ外再監査で固定する地形判定

- `ALT`（高山ツンドラ）は標高だけで森林・高山草原を上書きしない。森林限界の目安を緯度別に適用し、低緯度では3800m以上、中緯度では2800m以上、高緯度では1800m以上を最低条件とする。ヒマラヤの alpine shrub and meadows は原則 `ALG`（高山草原）から開始し、乾燥・高標高・緩起伏が揃うセルだけを `ALT` とする。
- `RPL`（岩石高原）は山岳系の汎用中間標高として使わない。WWFの乾燥・石質エコリージョン、またはサハラのハマダ、ゴビ、アラビア内陸、パタゴニア台地、豪州内陸など§5の実在アンカーに一致する場合だけ採用する。
- `PLT`（高原）は「標高が800m前後」という理由だけで採用しない。エチオピア高原、デカン高原、およびグレートプレーンズ西部High Plainsのように、広く高く緩やかな面が実在する地形に限定する。カザフ草原など単なる標高上昇は `STP`/`HST` で表す。
- `WET`（湿地）はWWFの氾濫原・湿地・沼沢エコリージョンと§6の明示的な大湿地に限定する。`MNG`（マングローブ）はWWFのマングローブエコリージョンかつ低標高熱帯・亜熱帯域に限定し、一般の熱帯海岸を一律変換しない。
- ニューギニアでは南北低地の湿地と沿岸マングローブを維持する一方、「ニューギニア高地」の監査範囲は中央山系へ限定し、低地湿地を高地レシピ違反として扱わない。
- アタカマ沿岸など、低地の乾燥被覆が急峻な山脈に直接接する海岸では、近傍DEMの大起伏だけを理由に `DSR`/`SHR` を山岳へ置換しない。セル自身のWWF被覆と標高を優先し、山地との勾配は隣接セル間の標高差で表す。

### 6.4 主要山脈の形状・可読性ルール

- 山脈は中心線をそのまま塗る「線状地形」にしてはならない。名称付き山脈の軸は検索範囲の指定にだけ使用し、実際の塗布範囲はETOPOの標高・局地起伏を満たす不規則な面から決定する。
- 山脈の太さは一定にしない。谷・盆地・峠を残し、標高と起伏に応じて山地林／亜高山林／山岳／高山岩稜／高峰岩壁／雪山・氷河へ段階的に遷移させる。
- アパラチア、ウラル、スカンディナビア山脈、大分水嶺など中低標高の古期山地を、標高アンカー2000mの `MTN` で無理に描かない。低部は `WDH`、高部は `MFR`/`SUB` とし、DEMが条件を満たす峰だけを上位山岳にする。
- ヒマラヤ、アンデス、ロッキーなどの大山系も一本の帯に統合せず、山麓植生・高原・乾燥盆地を残した複数幅の山岳面として表現する。
- 固有山頂の補正に円形・楕円形の半径スタンプを使わない。全球解像度で山頂を補う場合は、指定座標周辺でDEM標高と局地起伏が最大の1セルだけを上位山岳へ昇格する。`HIM` はエベレスト相当セルに限定する。
- 火山データベースの点も半径スタンプで拡張しない。各火山座標の近傍でDEM標高が最も高い適格陸セル1つだけを `VOL` にし、火山弧は複数火山セルの自然な集合として表す。
- 生成時には主要山脈ごとにDEM対象セル、山岳表現セル、被覆率、最大連続成分率を監査表へ出力する。低被覆または過度な分断は視覚確認対象とする。

### 6.5 人工形状を残さない全球監査

- 極域を固定緯度の水平帯で塗り分けない。南極の沿岸氷床と内陸氷冠高原は、実際の海岸線からの距離と低周波の不規則境界で分ける。
- 円・楕円・十字・一定半径ディスク、長い水平／垂直境界、同一小形状の反復を生成後に検査する。実地理または原資料形状で説明できないものは採用しない。
- 点データを表示する場合も、全球マップ上では原則1セルとする。面積を持つ地形へ拡張するにはDEM・被覆・原資料ポリゴンのいずれかによる根拠が必要。
- ETOPOの等深線が投影・縮小によって長い水平段差になる場合は、深度閾値へ低周波・小振幅の決定論的補正を加えて段差だけを崩す。海溝・深海・外洋・大陸棚という大域的な深度区分は変えない。
- WWFエコリージョン境界の水平・垂直なポリゴン辺が広域植生境界として露出した場合も、平原・森林・ジャングル・乾燥地系列の間だけを対象に1セル以内で不規則化する。山岳・高山・湿地・特殊乾燥地・海岸はこの後処理で変更しない。
- `PLR`極地砂漠が温帯森林・山林・平原へ直接接してはならない。原資料の縮小で直接接続が生じた場合は、間に1セルの`TND`ツンドラ遷移帯を置く。
- `TND`ツンドラも温帯森林・山林・平原へ直接接続させず、間に1セルの`CTG`寒冷タイガを置く。

## 7. データモデルと出力フォーマットの変更

### 7.1 コード体系（確定: 固定3文字ニーモニック）

現状フォーマットは **1セル=1文字**（[app.js](../app.js) の `rows: state.grid.map(row => row.join(""))`、読込は `row.split("")` かつ `row.length === size`）。そして**アルファベット26文字はすでに全て使用済み**（＋数字1〜4）。63 種にするには 1 文字コードが足りない。

**決定: 全コードを固定3文字・大文字ニーモニックにする**（例 `OCN`=海, `FOR`=森林, `HIM`=ヒマラヤ）。確定一覧は付録A。

- 容量: 3文字（26³=17,576）で 63 種は無論、将来の細分化にも枯渇しない。
- なぜ「固定長」か: 区切りなしの連結（羅列）で一意に分解するには全コードが同じ長さである必要がある。可変長（1〜3文字）を区切りなしで詰めると `"ABC"` が `A|B|C` か `AB|C` か `ABC` か判別できない。→ **固定3文字なら区切り不要のまま3文字ずつチャンクして分解できる**。
- 利点: マップJSONが自己文書的になる（`HIM` を見れば人間もヒマラヤと分かる）。
- 変更点:
  - 書き出し `rows: grid.map(row => row.join(""))` — グリッドが3文字トークンを持てば連結のままでよい（変更不要）。
  - 読み込み — `row.length === size` → `row.length === size*3`、`row.split("")` → **3文字ごとのチャンク**に置換。
  - 旧マップ移行 — 旧1文字コード→3文字の変換表（付録Aの「旧」列）を用意し、読込時に旧形式を検出して自動変換（`normalizeRows`／レガシー legend 検出の隣に追加）。
  - PNG入出力は色ベースのため影響なし。ファイルサイズは約3倍（同一トークンの繰り返しでgzipがほぼ相殺）。

### 7.2 凡例（legend）の拡張 — 3D生成器との契約

現状 [app.js](../app.js) `exportJson()` の凡例は `{ name, jp, color }` のみ。**3D生成器が地形と材質を決めるには標高等が必須**。凡例を次に拡張する。

```jsonc
"legend": {
  "HIM": {
    "name": "himalaya", "jp": "極高山（ヒマラヤ）", "color": "#ffffff",
    "category": "alpine",
    "elevation": 8500,        // 目標高度 m
    "temp": "cold",           // cold | temperate | hot
    "humidity": "dry",        // dry | mid | wet
    "relief": "peak"          // flat | rolling | rugged | peak（起伏の振幅）
  }
}
```

これにより、JSON 単体で（このツールを知らなくても）3D 生成器が高さ・起伏・植生材質を決定できる。

### 7.2.1 3D生成器向けルール: 浜 (`BCH`) の植生継承

> これは map_maker（本ツール）自体の仕様ではなく、**3D生成器が実装すべきルール**の申し送り。map_maker側のデータ・塗り分けは変更しない。

`BCH`(浜) は「陸と水の境界線」を表す帯であり、実際の地理では浜そのものが植生を持つのではなく、**背後の陸地の植生がそのまま渚際まで続いている**（熱帯の浜の背後は `rainforest`/`mangrove`、温帯の浜の背後は `plains`/`forest`、寒冷海岸の浜の背後は `tundra`/`ice_sheet`、というように浜の見た目は隣接陸バイオームに依存する）。

3D生成器は `BCH` セルを描画する際、次の規則で植生材質を決定する:

- **`BCH` に隣接する陸側（`coastal`/`OCN` 以外）のバイオームを1つ特定**し、その `category`（および `temp`/`humidity`）由来の植生材質を `BCH` セルにも適用する。渚のごく縁（数セル分）は砂地のまま残し、内陸側だけ植生を継承してよい。
- 複数の陸バイオームに隣接する場合は、**最も長く接している陸バイオーム**を採用する。
- 隣接陸が存在しない（全周を海/浜で囲まれた孤立浜など）場合のみ、`BCH` 自身の既定植生（温帯・裸地の砂浜）を使う。
- 標高・起伏（`elevation`=5, `relief`=flat）は `BCH` 自身の値をそのまま使う。継承するのは植生材質のみで、地形は変えない。

### 7.2.2 浜 (`BCH`) の塗り方（確定: 連続海浜方式）

案C相当の「主要な砂浜海岸を自然な連続区間として描く」方式を正式採用する。各海岸セルを独立乱数で点在させてはならない。

- 候補は外洋に上下左右または斜めで接する陸側1セル帯に限定する。内陸や海から2セル以上離れた場所へ `BCH` を置かない。
- ETOPO標高250m以下、周囲3×3の陸上起伏350m以下を目安とし、平原・サバンナ・ステップ・低木・砂漠・低地林・低地ジャングル・熱帯小島の海岸を候補にする。
- 岩石・断崖海岸、山岳海岸、マングローブ、湿地・高層湿原、ツンドラ、氷床・氷原・氷河では `BCH` を引かず、元のバイオームを水際まで延ばす。
- 固定シードの低周波連続場で候補海岸を区間単位に選ぶ。細かな場は区間端の揺らぎにだけ使い、独立した点状ノイズを作らない。
- 適格海岸上の1セル間隔は補間して連結し、2セル未満の孤立浜は除去する。ただし1セル陸塊の熱帯小島は全体を浜として残してよい。
- 出力上の浜は1セル厚を基本とし、海岸線そのものを変更しない。§7.2.1の隣接陸バイオーム植生継承は引き続き適用する。
- 縮小版PNG／JSONでは最近傍縮小後に海隣接条件と連続性を再検査し、海から離れた浜と孤立1セル浜を周辺の陸バイオームへ戻す。

### 7.3 `BIOMES` 定義への追加フィールド

§1 の「特徴と標高」拡張（実施済み）に `relief` を追加する。最終フィールドは:
`code, name, jp, color, category, elevation, temp, humidity, relief, note`。
